蝴蝶效应：从“给女儿起名”开始的离谱展开

如果我说，这个博客的诞生，最初只是为了给我还没出生的女儿起个名字，你敢信？

几天前，我打开 AI 对话框，本意是找这位 Gemini “算命大师”算个吉利名字。结果聊着聊着，话题像脱缰的野马——从五行八字聊到了 n8n 工作流，又从自动化聊到了“我该把这些折腾记录下来”。

于是，一个致命的问题抛了出来：“写博客用什么程序好？”

这就是一切灾难（和重生）的开始。

第一阶段：死磕 WordPress，越陷越深

我和 Gemini 的第一回合，选择了看似稳妥的 WordPress。
部署过程很顺，但问题很快暴露：慢。
毕竟这是一台ubuntu16内核我忘了是多少，连docker都不能跑，不支持PHP7.3的老旧Chicago VPS服务器，线路还非常一般。
为了解决慢，我开始了一系列“赌徒式”的加码：

  1. 上 CDN：为了让它快一点，我决定接入 Cloudflare。
  2. “冤大头”式迁移：为了配合 CF 的解析，我咬牙把域名注册商迁了过去。为此，我不得不含泪多交了一年的域名续费钱（这可是真金白银啊！）。
  3. 无底洞优化：钱都花了，必须得快吧？结果还是不行。我又装静态化插件、魔改 WP 主题、调整 PHP 参数……

整整一天，我没写半个字，全在对着 itdog和tcptest发呆。

第二阶段：守着“传家宝”服务器的倔强

为了给这个慢到令人发指的博客找个理由，我跟 Gemini 摊牌了：

这台服务器是当年的“传家宝”，虽然是 OpenVZ 的老旧架构，跑的是 Ubuntu 16 这种古董系统，但它有一个足以让我忽略所有缺点的神仙配置——它有 5 个公网 IP！

我就像一个守着破烂不肯扔的守财奴，哪怕它性能拉胯，哪怕它系统老旧，但我就是觉得“5 个 IP 扔了太可惜”。

Gemini 当时也被我整无语了，它只说了一句直击灵魂的话：

“这台服务器一年才70 块钱。为了这 70 块，你打算忍受一辈子的 3 秒加载速度吗？”

我看着那台拥有 5 个 IP 却跑不动博客的 Chicago VPS，第一次觉得它像个鸡肋——食之无味，弃之可惜。
当然我还有个RackNerd的10美元年付服务器，但是这个服务器一方面性能一般，最重要的是它在跑n8n，速度呢，虽然不快，但毕竟是直连，洛杉矶DC03的服务器，国内去回双程都是直连，速度尚可，但是我不想破坏它的环境。

第三阶段：一张备案域名的“降维打击”

转折点出现在我无意中说漏嘴的一句话：

“其实……我还有一个备案过的域名。”

那一刻，我仿佛看到了 Gemini 顺着网线拍大腿：“有备案域名你不早说？！”

所有的逻辑瞬间推倒重来。什么 Cloudflare，什么美国 VPS，统统闪开。
我们确立了最终的“白嫖 + 极速”战略：

  1. 放弃动态：彻底抛弃笨重的 WordPress，改用 Hexo。
  2. 国内开挂：因为有备案，直接部署到又拍云（UPYUN）。利用国内边缘节点，实现物理层面的秒开。
  3. 海外兜底：利用 Github Pages 覆盖海外，阿里云 DNS 做智能分流。

第四阶段：为了“懒”，我们必须解耦

虽然速度解决了，但我还有一个巨大的痛点：我电脑太多了。

公司三台电脑、家里台式机、随身笔记本、父母家的电脑、外地办公室的电脑……如果用传统的 Hexo 部署方式，我得在每一台电脑上都装 Node.js、装 Git、拉取源码、同步主题配置。
“不想每个电脑都装 Hexo”，成了压垮骆驼的最后一根稻草。

于是，Gemini 帮我设计了一套前后端彻底解耦的方案，顺便把我的 VPS 利用了起来：

  • 后台工厂（Private Repo）：用来存放 Hexo 源码。
    • 痛点解决：我再也不用在本地跑 hexo g 了！
    • 随时随地：我在任何一台电脑（甚至手机）上，只要能把 Markdown 文件推送到 GitHub，或者直接在 VPS 的 Code-Server 网页上写，剩下的事全交给云端。
  • RackNerd (苦力)：那台 RackNerd 服务器也没闲着，作为生产车间跑 n8n 抓取数据，生成的文章自动推送到仓库。
  • 自动化流水线：GitHub Actions 负责编译，自动把成品分发到又拍云和 GitHub。

结局：0.3 秒的救赎

当我在测速网站上看到全球各地全是绿色的 0.3s 时，我意识到，之前的折腾终于结束了。

从为了省钱而多花钱，到为了 5 个 IP 守着 OpenVZ 老服务器，再到为了“多电脑不装环境”而实现的彻底解耦。
如果没有 Gemini 全程陪跑——不仅教我改 Nginx、配数据库、修主题，还要听我关于“传家宝”的碎碎念——我可能还在对着 WordPress 的后台发愁。

最后，这位居功甚伟的 AI 伙伴给了我最后一条建议，也是最重要的一条：

“架构已经完美了。内容为主，少折腾。去写吧。”

是啊，别修管道了，水已经来了。
